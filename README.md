# AI Pricing Demo

Standalone local pricing/checkout app for the Lago assignment. It demonstrates:

- plan selection (`Standard` €20/month, `Premium` €50/month)
- customer signup with billing address
- Stripe card registration through Lago's hosted checkout URL
- subscription creation after payment-method verification
- metered token event ingestion
- invoice listing for the signed-up customer

## Prerequisites

1. Lago running locally via Docker Compose in the repository root:
   ```bash
   docker compose up -d
   ```
2. A Lago organization created at `http://localhost` and an API key copied from **Developers**.
3. Stripe connected in Lago (**Settings → Integrations → Stripe**) using your sandbox keys.
4. Note the Stripe payment provider **code** shown in Lago (for example `teststripe`).
5. Set the Stripe integration **Success redirect URL** to:
   `http://localhost:3001/checkout/complete?session_id={CHECKOUT_SESSION_ID}`
   Without this, Stripe will not return the customer to the app after card registration, and the subscription will not be created.

## Setup

```bash
cd apps/ai-pricing-demo
cp .env.example .env.local
```

Edit `.env.local`:

- `LAGO_API_KEY`: your Lago organization API key
- `LAGO_PAYMENT_PROVIDER_CODE`: the Stripe provider code configured in Lago
- `SESSION_SECRET`: any long random string

Install dependencies and configure Lago objects:

```bash
npm install
npm run setup:lago
```

This creates or updates:

- billable metrics: `easy_tokens`, `medium_tokens`, `pro_tokens`
- plans: `standard_plan`, `premium_plan`
- per-model monthly allowances and overage prices

## Run locally

```bash
npm run dev
```

Open `http://localhost:3001`.

## Demo flow

1. Choose **Standard** or **Premium** on the pricing page.
2. Fill in first name, last name, personal/company details, and billing address.
3. Continue to Stripe and register a test card (`4111 1111 1111 1111`).
4. Stripe redirects back to `/checkout/complete`, which:
   - links the registered card to the Stripe/Lago customer
   - creates the Lago subscription
   - charges the generated invoice until `payment_status=succeeded`
5. You are then redirected to the Lago **customer portal**.
6. Open `/usage?customer=<external_id>` to simulate metered token events for a selected subscription and model.
7. Open `/portal?customer=<external_id>` (or use **Customer portal** in the nav) to access the Lago customer portal.

## Billing rules implemented

| Plan | Monthly fee | Included models | Allowance per model | Overage |
| --- | --- | --- | --- | --- |
| Standard | €20 | Easy, Medium | 1,000,000 tokens | Easy €0.000001, Medium €0.000002 |
| Premium | €50 | Easy, Medium, Pro | 5,000,000 tokens | Easy €0.000001, Medium €0.000002, Pro €0.000003 |

The app blocks usage events for models not included in the selected plan.

## Scripts

- `npm run dev` — start the app on port `3001`
- `npm run setup:lago` — create/update Lago metrics and plans
- `npm run test` — run unit tests
- `npm run lint` — run ESLint

## Security notes

- Never commit `.env.local`.
- Never put the Lago API key or Stripe secret key in client-side code.
- Card data is collected only on Stripe's hosted checkout page generated by Lago.
